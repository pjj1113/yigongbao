# 首页看板 — 后端插入数据通用文档

## 接口

```
GET /dashboard/{roleCode}
```

`roleCode` 取值：`salesman` / `area_manager` / `designer` / `design_manager` / `production` / `production_manager` / `quality` / `warehouse` / `finance` / `company_admin` / `super_admin`

---

## 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `timeRange` | string | 是 | `today` / `week` / `month` / `quarter` / `year` / `custom` |
| `startTime` | string | custom时必填 | `"2026-05-01 00:00:00"` |
| `endTime` | string | custom时必填 | 同上格式 |

---

## 响应结构

```json
{
  "cards": [ CardVO ],
  "charts": [ ChartVO ],
  "todos": [ TodoVO ],
  "system": SystemVO   // 仅 super_admin
}
```

---

## 数据结构

### CardVO

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `key` | string | 是 | 唯一标识，见各角色 key 表 |
| `title` | string | 是 | 卡片标题 |
| `value` | number \| string | 是 | 主数值，如 `156` 或 `"92.3%"` |
| `unit` | string | 否 | 单位，如 `"单"` `"元"` `"天"` |
| `change` | string | 否 | 环比文字或补充说明，如 `"+12.5%"` `"通过 20 / 返工 3"` |
| `trend` | string | 否 | `up` / `down` / `flat`（up=红，down=绿） |

### ChartVO

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `key` | string | 是 | 唯一标识 |
| `title` | string | 是 | 图表标题 |
| `type` | string | 是 | `line` / `bar` / `pie` / `ring` / `funnel` / `heatmap` |
| `data` | object | 是 | 见下方各图表类型 |

---

#### 折线图 (`type: "line"`)

用于展示随时间变化的趋势，支持多系列叠加。

| 字段 | 类型 | 说明 |
|------|------|------|
| `xAxis` | string[] | X 轴标签，通常为日期，如 `["5/21", "5/22", "5/23"]` |
| `series` | array | 数据系列列表，每项含 `name`（系列名）和 `data`（数值数组，长度必须与 `xAxis` 一致） |

```json
{
  "xAxis": ["5/21", "5/22", "5/23", "5/24", "5/25", "5/26", "5/27"],
  "series": [
    { "name": "新建订单", "data": [12, 15, 18, 14, 20, 16, 22] },
    { "name": "完成订单", "data": [8, 10, 12, 11, 15, 13, 18] }
  ]
}
```

---

#### 柱状图 (`type: "bar"`)

用于对比不同分类（人员、部门、阶段等）的数值，支持多系列分组。

| 字段 | 类型 | 说明 |
|------|------|------|
| `xAxis` | string[] | X 轴标签，如人名、部门名、阶段名 |
| `series` | array | 数据系列列表，结构同折线图 |

```json
{
  "xAxis": ["张三", "李四", "王五", "赵六"],
  "series": [
    { "name": "生产中", "data": [10, 12, 11, 13] },
    { "name": "已完成", "data": [32, 28, 35, 30] }
  ]
}
```

---

#### 饼图 (`type: "pie"`) / 环形图 (`type: "ring"`)

用于展示各分类占比，`ring` 为中空环形样式，数据结构相同。

| 字段 | 类型 | 说明 |
|------|------|------|
| `items` | array | 每项含 `name`（分类名）和 `value`（数值） |

```json
{
  "items": [
    { "name": "待审核", "value": 8 },
    { "name": "设计中", "value": 15 },
    { "name": "生产中", "value": 12 },
    { "name": "已完成", "value": 106 }
  ]
}
```

---

#### 漏斗图 (`type: "funnel"`)

用于展示流程各阶段的数量递减，`items` 按流程顺序排列，从大到小。

| 字段 | 类型 | 说明 |
|------|------|------|
| `items` | array | 每项含 `name`（阶段名）和 `value`（数值），按流程顺序排列 |

```json
{
  "items": [
    { "name": "创建", "value": 500 },
    { "name": "审核通过", "value": 460 },
    { "name": "设计完成", "value": 420 },
    { "name": "生产完成", "value": 390 },
    { "name": "质检通过", "value": 370 },
    { "name": "已出库", "value": 350 }
  ]
}
```

---

#### 热力图 (`type: "heatmap"`，仅 super_admin)

用于展示二维矩阵数据，如用户在一周各时段的活跃度分布。

| 字段 | 类型 | 说明 |
|------|------|------|
| `xAxis` | string[] | X 轴标签，表示时间刻度，如 `["00:00", "04:00", "08:00", "12:00", "16:00", "20:00"]` |
| `yAxis` | string[] | Y 轴标签，表示星期，从下到上排列：`["周日", "周六", "周五", "周四", "周三", "周二", "周一"]` |
| `items` | array | 三元组列表，每项为 `[xIndex, yIndex, value]`，索引对应 `xAxis`/`yAxis` 下标，`value` 为该格热度值 |

```json
{
  "xAxis": ["00:00", "04:00", "08:00", "12:00", "16:00", "20:00"],
  "yAxis": ["周日", "周六", "周五", "周四", "周三", "周二", "周一"],
  "items": [
    [0, 0, 15], [1, 0, 22], [2, 0, 45], [3, 0, 68], [4, 0, 52], [5, 0, 28],
    [0, 1, 12], [1, 1, 18], [2, 1, 38], [3, 1, 55], [4, 1, 45], [5, 1, 22]
  ]
}
```

### TodoVO

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | number | 是 | |
| `title` | string | 是 | 待办描述 |
| `count` | number | 否 | 数量 |
| `link` | string | 否 | 前端跳转路由 |
| `urgent` | boolean | 否 | 是否加急，默认 false |

### SystemVO（仅 super_admin）

| 字段 | 类型 | 说明 |
|------|------|------|
| `healthStatus` | string | `healthy` / `warning` / `error` |
| `avgResponseTime` | string | 如 `"120ms"` |
| `onlineUsers` | number | 当前在线用户数 |
| `avgOrderCycle` | string | 如 `"432h"` |

---


## 注意事项

1. **`change` 字段复用**：无 `trend` 时不显示箭头，可用于补充说明文字（如 `"通过 20 / 返工 3"`）。
2. **`value` 类型**：百分比类卡片（如 `reviewPassRate`、`completionRate`）返回字符串 `"91%"`，数值类返回 number。
3. **series 与 xAxis 等长**：折线图/柱状图每个 series 的 `data` 数组长度必须与 `xAxis` 一致。
4. **heatmap items**：三元组 `[xIndex, yIndex, value]`，索引对应 `xAxis`/`yAxis` 数组下标。
